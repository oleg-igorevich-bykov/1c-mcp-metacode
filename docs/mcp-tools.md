# Инструменты MCP

Справочник инструментов (tools), которые публикует MCP-сервер. Это API-контракт для внешних агентов и
IDE. Здесь — что делает каждый инструмент и когда он доступен; подробные сценарии поиска вынесены в
[search.md](search.md), [bsl-code-search.md](bsl-code-search.md) и [object-summary.md](object-summary.md).

## Подключение

MCP-сервер доступен по адресу `http://<хост>:<порт>/mcp` (порт задаётся под проект, путь — `MCP_PATH`).

- По умолчанию транспорт **streamable-http**.
- При `MCP_USE_SSE=true` — режим **SSE**.

Подключается любой MCP-совместимый клиент. Пример конфигурации:

```json
{
  "mcpServers": {
    "1c-metacode": {
      "url": "http://localhost:6001/mcp",
      "type": "streamable-http",
      "timeout": 300
    }
  }
}
```

## Доступность инструментов

Часть инструментов появляется только при включённых флагах загрузки/подсистем — если
соответствующие данные не построены, инструмент не публикуется. Кроме того, видимость любого
инструмента можно вручную включать и отключать из веб-консоли (см. [web-console.md](web-console.md)),
не меняя конфигурацию.

## Инструменты, доступные всегда

Работают на базовом графе метаданных.

| Инструмент | Назначение |
|-----------|------------|
| `get_metadata` | получить объект(ы) метаданных по имени/идентификатору с их свойствами |
| `find_metadata_objects` | найти объекты метаданных — по структурным признакам или по описанию (см. [search.md](search.md)) |
| `get_metadata_object_structure` | получить состав объекта: реквизиты, табличные части, ресурсы, измерения, формы, команды |
| `find_metadata_elements` | найти элементы внутри объектов (реквизиты, ресурсы, измерения, команды и т.д.) |
| `find_metadata_usages` | найти, где объект используется (в реквизитах, элементах форм, движениях, правах) |
| `get_metadata_element_type` | получить тип конкретного элемента (реквизита/ресурса/измерения) |
| `find_predefined_values` | найти предопределённые значения объекта |
| `get_access_rights` | получить права ролей на объект (кто и какой доступ имеет) |
| `get_metadata_details` | разрешить ссылку на узел (объект, дочерний элемент, форма, элемент/событие формы, команда, GUID, routine_id) и получить его отфильтрованные свойства (режимы `resolve` / `properties`) |
| `inspect_metadata_object` | обзорная карточка объекта: что по нему есть в графе — счётчики и флаги по секциям (состав, формы, использования, зависимости, права, код), опционально с краткими списками по секциям |
| `get_form_structure` | получить структуру формы: элементы, реквизиты, события, команды и их обработчики |
| `find_form_links` | найти связи формы (привязки элементов к данным и командам) |
| `get_event_subscriptions` | получить подписки на события и их обработчики |
| `find_dependency_paths` | найти пути зависимостей между объектами по графу связей |
| `get_tool_return_schema` | получить документированную схему ответа указанного инструмента |

## Инструменты, зависящие от флагов

| Инструмент | Требует флаг | Назначение |
|-----------|--------------|------------|
| `search_bsl_routines` | `LOAD_BSL_SIGNATURES` | поиск процедур/функций по имени, сигнатуре или описанию (см. [search.md](search.md)) |
| `get_bsl_routine_body` | `LOAD_BSL_SIGNATURES` | получить тело процедуры/функции |
| `get_bsl_modules` | `LOAD_BSL_SIGNATURES` | получить модули объекта и их состав |
| `get_bsl_call_graph` | `LOAD_BSL_SIGNATURES` | получить граф вызовов вокруг процедуры (кто вызывает / что вызывается) |
| `search_bsl_code` | `ENABLE_BSL_CODE_SEARCH` (+ `LOAD_BSL_SIGNATURES`) | семантический поиск по телу кода BSL (см. [bsl-code-search.md](bsl-code-search.md)) |
| `find_objects_by_summary` | `OBJECT_SUMMARY_ENABLED` | поиск объектов по сгенерированной сводке (см. [object-summary.md](object-summary.md)) |
| `get_extension_object_diff` | `LOAD_EXTENSIONS` | сравнение объекта расширения с базовым (см. [extensions.md](extensions.md)) |

## Формат ответа

Формат итогового ответа задаётся `RESPONSE_FORMAT`:

- `text` — человекочитаемый текст;
- `json` — структурированный JSON;
- `toon` — компактный формат, экономящий токены.

Дополнительно объём ответа сокращают опции сжатия (`RESPONSE_COMPACT_NODES`, `RESPONSE_COMPACT_REFS`,
`RESPONSE_JSON_COMPACT`): убирают служебные поля и заменяют повторяющиеся ссылки короткими маркерами.

Чтобы заранее узнать структуру ответа конкретного инструмента, вызовите `get_tool_return_schema`.

## Лимиты и пагинация

- `QUERY_DEFAULT_LIMIT` — лимит по умолчанию, если клиент не указал свой.
- `QUERY_MAX_RESULTS` — жёсткий потолок числа строк в ответе (превышение обрезается).
- Отдельные лимиты по умолчанию есть у поиска по описанию и по коду (см. соответствующие документы).

Результаты, где это применимо, поддерживают пагинацию — инструмент возвращает признак/курсор для
получения следующей порции.

## Мультипроект

Указывать проект в вызовах не нужно: сессия знает свой `PROJECT_NAME` и связанные проекты
(`RELATED_PROJECTS`) и фильтрует результаты автоматически.
